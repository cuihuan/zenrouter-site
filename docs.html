Documentation

Everything you need to route your first request through ZenRouter. 从零接入 ZenRouter 的完整指南。

1. Get an API key · 获取 API Key

  1. Sign up at api.zenrouter.io and verify your email.
  2. Buy a credit pack on the Top-up page (one-time purchase, never recurring).
  3. Open API Keys in the console and create a key. Keep it secret — it spends your balance.

2. The only change you need: base URL

ZenRouter exposes an OpenAI-compatible surface. Any client that lets you change the base URL works as-is:

Client typeSettingValue
OpenAI-style SDKs & toolsbase_url / OPENAI_BASE_URLhttps://api.zenrouter.io/v1
Anthropic-style tools (Claude Code)ANTHROPIC_BASE_URLhttps://api.zenrouter.io

3. Claude Code

# Add to ~/.zshrc or ~/.bashrc, then restart your shell
export ANTHROPIC_BASE_URL=https://api.zenrouter.io
export ANTHROPIC_AUTH_TOKEN=your-zenrouter-key

claude

Claude Code now sends all model traffic through your ZenRouter balance. To switch back, remove the two variables.

4. Codex CLI

# ~/.codex/config.toml
model = "gpt-5.5"

codex

5. OpenAI SDK (Python / Node)

# Python — pip install openai
from openai import OpenAI

client = OpenAI(
    base_url="https://api.zenrouter.io/v1",
    api_key="your-zenrouter-key",
)
resp = client.chat.completions.create(
    model="claude-sonnet-4-6",
    messages=[{"role": "user", "content": "Hello"}],
)
print(resp.choices[0].message.content)
// Node — npm install openai
import OpenAI from "openai";

const client = new OpenAI({
  baseURL: "https://api.zenrouter.io/v1",
  apiKey: process.env.ZENROUTER_API_KEY,
});

7. Models & routing · 模型与路由

Use the model IDs listed in your dashboard (Models page) — they follow each provider's familiar naming. Requests are routed across redundant upstream channels; if one channel degrades, traffic fails over automatically.

The available model list and per-token rates are maintained live in the console and may change as providers ship new models. 模型列表与单价以控制台实时数据为准。

8. Billing · 计费说明

  • Prepaid credits. You buy a credit pack once; there is no subscription and no automatic charge.
  • Per-token metering. Each request deducts credits based on input + output tokens at the per-model rate shown in the console.
  • Full audit trail. Every request appears in your usage log with token counts and cost.
  • Spend controls. Set per-key quotas to cap how much any single key can spend.
  • Refunds. Unused credits are refundable within 7 days of purchase — see the Refund Policy.
